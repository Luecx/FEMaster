DSL

DSL Reference

The FEMaster input deck language, grouped by modelling workflow. Commands start with *; keywords appear on the command line; data follows in comma-separated rows.

Command groups

Model

Nodes, elements, surfaces, and named sets — the geometric backbone.

Properties

Materials, sections, profiles, orientations, fields, and point masses.

Loads & supports

Collector-based boundary data activated by loadcases.

Constraints

Coupling, tie, contact, connector, and rigid-body helpers.

Analysis

Loadcase blocks, solver controls, topology, transient, and diagnostics.

Deck structure

** Comments start with **
*COMMAND, KEYWORD=VALUE
dataline1_field1, dataline1_field2
dataline2_field1, dataline2_field2

Some commands open a context — notably *MATERIAL and *LOADCASE — meaning that subsequent property or control commands are nested inside them until a new top-level command is encountered.
