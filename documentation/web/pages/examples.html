Examples

Starter patterns and debugging strategies for common analyses.

Static sanity check

Start every new model with LINEARSTATIC, DIRECT solver and NULLSPACE check. Request stiffness output if the model appears singular or over-constrained — it will reveal which DOFs are missing constraints.

LINEARSTATIC DIRECT REQUESTSTIFFNESS

Contact debug

Use NONLINEARSTATIC with small increments and a moderate penalty. Explicitly verify contact normals before scaling up. CONSTRAINTSUMMARY prints generated contact equations without changing the solve.

NONLINEARSTATIC CONTACT CONSTRAINTSUMMARY

Buckling preload

Always validate the static preload loadcase first. Then run LINEARBUCKLING with NUMEIGENVALUES and, if needed, a shift via SIGMA to target a specific eigenvalue range.

LINEARBUCKLING NUMEIGENVALUES SIGMA

Transient response

Define amplitudes and time controls before increasing model size. Set Newmark parameters explicitly. Control output volume with WRITEEVERY — dense output on a large model can produce very large .res files.

TRANSIENT AMPLITUDE NEWMARK WRITEEVERY
